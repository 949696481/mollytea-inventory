# 库存登记 · 使用与部署说明

一个共享后端的网页 App(PWA)——你(管理员)和员工用同一个网址登录,各自的
账号决定能看到/能改什么。每天输入物品的库存盘点数值,自动算出比昨天消耗
了多少、花了多少钱,数据都存在同一份 Google Sheet 里,任何设备打开都是同
一份实时数据。

---

## 第一步:部署 Google Sheet 后端(约 5 分钟,只需要做一次)

1. 新建一个 Google Sheet(专门给这个 App 用,不要跟别的表格混用)。
2. 菜单栏 **扩展程序 → Apps Script**。
3. 把 `google-apps-script.gs` 的全部内容复制粘贴进去,覆盖默认代码。
4. 右上角 **部署 → 新建部署**:
   - 类型选 **网页应用 (Web app)**
   - **执行身份**:我 (Me)
   - **谁可以访问**:任何人 (Anyone)
5. 部署后会给一个网址,形如 `https://script.google.com/macros/s/xxxx/exec`。
6. 打开 `app/config.js`,把这个网址填进 `API_BASE_URL`。
7. （建议)回到 Apps Script 编辑器,选中 `installDailyCleanupTrigger` 函数,
   点"运行"一次——装一个每天清理过期登录会话的定时任务,免得 `sessions`
   表越滚越大。只需要点这一次。

这一整份表格现在装下**所有门店**的数据(不再是一个门店一份独立表格),
`stores`/`categories`/`users`/`sessions` 是全局的,每个分类还会自动生成
自己的 `fields_<id>`/`items_<id>`/`log_<id>` 三张工作表。

---

## 第二步(可选,建议):迁移已有的桌面版数据

如果你之前已经在桌面版(`D:\DevTools\InventoryDesktopApp`)里录过真实数据
(门店、分类、字段、物品、历史记录、员工账号),可以用 `migrate_local_data.py`
一次性搬过来:

```
python migrate_local_data.py
```

会依次提示你粘贴新后端网址、给管理员和每个员工账号设置新密码(旧密码的哈希
算法跟新后端不一样,没法直接沿用,每个账号都需要一个新密码——迁移完告诉
对应的人就行)。只读本地 CSV,不会修改/删除任何本地文件。

如果是全新开始,跳过这一步,直接打开 App 走"设置管理员账号"就行。

---

## 第三步:发布到 GitHub Pages(这样员工用自己的电脑就能打开)

1. `gh auth login`(需要你自己在浏览器里授权一次)。
2. 新建或复用一个 GitHub 仓库,把这整个文件夹(根目录的 `index.html` 是
   欢迎页,`app/` 子文件夹是真正的 App)推上去。
3. 仓库 **Settings → Pages → Source** 选对应分支,保存。
4. 几分钟后就能通过 `https://你的用户名.github.io/仓库名/` 访问——这是
   给员工的那个网址,不需要她额外安装任何东西。

本地测试(不用等 GitHub Pages)可以先跑:
```
python -m http.server 8000
```
然后浏览器打开 `http://localhost:8000/`。

---

## 第四步:账号与权限

- 第一次打开网址,会看到"设置管理员账号"——那就是你自己的登录账号。
- 登录后,店铺选择页右上角有个"账号"按钮,进去给员工建账号:填姓名/
  用户名/密码,选她所属的门店。员工登录后会直接跳过门店选择,进到她自己
  门店的分类页,而且看不到"管理"/"管理字段"/"物品设置"/改结算货币这些
  配置类按钮——那些操作后端本身也会拒绝非管理员账号,不是只靠前端隐藏。
- 忘记员工密码可以在"账号"页里"重置密码"。

---

## 第五步:日常使用

1. 选门店 → 选分类(比如"奶制品")。
2. "目前总库存"点一个物品的方块,或者在"今天的记录"表格里填数字。
3. 有配置"库存盘点"字段的分类,点"计算今日消耗"看一眼预览,确认没问题
   再"保存记录";没配置的分类直接"保存"。
4. "最近记录"点进某一天能看到当天消耗总结,点某一条能修正错误的数字。

在 Chrome / Edge 里打开 `app/` 那个网址后,地址栏会出现"安装"图标,点一下
就能把它装成桌面/主屏幕上的独立 App 窗口。

## 文件说明

| 文件/文件夹 | 作用 |
|---|---|
| `index.html`(根目录) | 欢迎页,一个按钮进 `app/` |
| `app/index.html` | App 主体(界面 + 计算逻辑) |
| `app/config.js` | 填 `API_BASE_URL` 的地方——换后端部署只改这一行 |
| `app/manifest.json` | PWA 配置(图标、名称、启动方式) |
| `app/sw.js` | 离线缓存支持(改过 `app/index.html` 记得把 `CACHE_NAME` 加一个版本号) |
| `app/icon-192.png` / `app/icon-512.png` | App 图标 |
| `google-apps-script.gs` | 粘贴进 Google Sheet 的后端代码 |
| `migrate_local_data.py` | 从桌面版本地 CSV 一次性搬数据的脚本(可选) |
